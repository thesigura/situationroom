## Phase 3 — Automated Intelligence Monitoring System

## Data to Capture
1. **Posts** from monitored accounts
2. **Replies** (often where analysts reveal conviction/uncertainty)
3. **Reposts/quote posts** (network propagation)
4. **High-engagement comments** (crowd narrative + expert pushback)

## Signal Extraction Pipeline
1. **Ingestion:** Pull fresh content every 10–20 minutes.
2. **Normalization:** Clean text, expand links, detect language, remove spam.
3. **Enrichment:**
   - Named entities (countries, companies, ports, shipping lanes, commodities)
   - Event tags (strike, sanctions, blockade, export control, outage)
   - Asset tags (Brent, WTI, copper, LNG, semis, freight indices)
4. **Sentiment and stance:**
   - General polarity (risk-on/risk-off)
   - Domain stance (bullish oil, bearish growth, escalation risk ↑)
5. **Topic clustering:** Dynamic clustering of posts into narratives.
6. **Anomaly detection:**
   - Repost spike vs 30-day baseline
   - Multi-analyst convergence on same event
   - Narrative vector shift vs previous week
7. **Scoring:** Assign signal confidence and investor relevance.

## Detection Logic
- **Unusual repost spikes:** z-score of repost velocity by account/topic.
- **Convergence detection:** N distinct high-credibility accounts mention same event in T minutes.
- **Narrative shift:** Embedding centroid drift for topic cluster beyond threshold.
- **Escalation laddering:** Sequence detection (exercise → warning → interception → strike).

## Phase 4 — Investor Intelligence Analysis Layer

## Event-to-Market Translation Rules

### Oil markets
- **Signals:** Strait/chokepoint threats, OPEC rhetoric change, refinery outages, sanctions tightening.
- **Implications:** Crude risk premium, backwardation changes, tanker rates, crack spreads, energy equities beta.

### Commodity markets
- **Signals:** Mine disruptions, export bans, weather + conflict overlap, shipping reroutes.
- **Implications:** Spot squeezes, curve steepening, basis volatility, cross-commodity substitution.

### Semiconductor supply chains
- **Signals:** Export-control shifts, fab outages, Taiwan military tensions, shipping/air cargo disruptions.
- **Implications:** Lead time extension, inventory repricing, margin pressure in hardware supply chains.

### Taiwan/China escalation
- **Signals:** PLA air/naval tempo spikes, ADIZ patterns, blockade rehearsal indicators, diplomatic breakdowns.
- **Implications:** TSMC risk premium, regional FX volatility, insurers/logistics repricing.

### Maritime disruptions
- **Signals:** Chokepoint incidents (Hormuz, Bab el-Mandeb, Malacca, SCS), insurance warnings, rerouting.
- **Implications:** Freight spike, delivery delays, inflation impulse via transport costs.

## Output Flags
- **Early warning (green/amber/red)**
- **Likely market transmission channels**
- **Probability market is underpricing event (low/medium/high)**

## Phase 6 — Technical Architecture (<$25/month)

## Recommended Stack
- **Language:** Python 3.11+
- **Orchestration:** cron (free) or GitHub Actions (low usage)
- **Scraping/collection:**
  - Primary: `snscrape` (if feasible for public data)
  - Backup: X API basic tier (only if needed; keep request volume minimal)
- **Storage:**
  - SQLite/Postgres Lite for raw posts + metadata
  - DuckDB for analytics queries
- **Vector DB:** Chroma (local, free) or FAISS
- **NLP/Embeddings:** sentence-transformers (local, free)
- **Sentiment/stance:** FinBERT or lightweight transformer classifier
- **Topic clustering:** BERTopic or HDBSCAN + embeddings
- **LLM summarization:** low-cost API model with strict token budgets (or local 7B/8B model)
- **Reporting:** Jinja2 templates + Google Docs API
- **Visualization (optional):** Streamlit local dashboard

## Low-Cost Deployment Options
1. **Personal machine only:** $0 recurring + optional API spend.
2. **Low-cost VPS (2 vCPU / 4GB):** ~$5–10/month.
3. **LLM API budget cap:** ~$10–15/month via daily token limits.

**Target monthly cost example:**
- VPS: $8
- LLM API: $12
- Misc (storage/alerts): $3
- **Total: ~$23/month**

## Data Pipeline Design
1. **collector.py**
   - Pull posts/replies/reposts/comments for watchlist
   - Store raw JSON + normalized text
2. **enricher.py**
   - Entity extraction, asset tagging, geo tagging
   - Sentiment/stance inference
3. **clusterer.py**
   - Daily narrative clustering
   - Cluster label generation and drift tracking
4. **detector.py**
   - Spike/convergence/shift rules
   - Assign confidence + urgency
5. **analyst.py**
   - Translate events → market implications via rule engine + LLM summary
6. **reporter.py**
   - Render daily report
   - Push to Google Docs

## Suggested Database Schema (minimal)
- `accounts(id, handle, category, credibility_score)`
- `posts(id, account_id, ts, text, engagement, type, url)`
- `entities(post_id, entity, type)`
- `signals(id, ts, signal_type, topic, confidence, urgency, summary)`
- `market_links(signal_id, asset, direction, horizon, rationale)`
- `narratives(id, day, topic_label, centroid_vector, drift_score)`

## Quality Controls
- Source credibility weighting (official/institutional > anonymous)
- Duplicate suppression (same headline reposted widely)
- Hallucination-safe summarization (quote source posts in report)
- Backtesting loop: compare detected signals vs subsequent asset moves

## Practical Tips to Maximize Signal
- Maintain a **tiered account list**:
  - Tier 1: high-credibility primary sources
  - Tier 2: fast OSINT sources
  - Tier 3: narrative interpreters
- Require **2-source confirmation** for high-impact alerts unless source is official.
- Track **who was early historically** and increase their score automatically.
- Keep reports concise; focus on **what changed**, **why it matters**, and **what may be mispriced**.
